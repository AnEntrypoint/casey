gm-method lesson: a user-reported defect is a HYPOTHESIS, not a spec. Witness the suspected layer in isolation against real services FIRST (here: dedup was already correct), which re-frames the real defect (a downstream dead-end stall the user perceived as the reported symptom). Process-of-elimination against real input beats believing the report's literal framing. Also: a degraded/fallback path must never dead-end -- have it advance the task by one step, guarded once-only by a durable append-only marker so it cannot nag or be clobbered by a concurrent writer.
