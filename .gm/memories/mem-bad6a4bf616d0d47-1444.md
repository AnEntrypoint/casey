casey (animal-disease surveillance chat agent) design detail: no hand-curated interpretation stand-ins. Where casey once carried a hand-curated lookup table as a stand-in for agent interpretation (a static SA-town gazetteer for map geocoding), the table was removed entirely and caseSystemPrompt instead trusts/instructs the model to use its own knowledge/judgment. Two protected classes of deterministic layer stay regardless and are NOT interpretation stand-ins: (1) the STOP/HUMAN opt-out/handoff safety keyword layer (detectContactIntent in gateway-hooks.js) -- must work even with the LLM down, backstops but does not replace the agent's own case_stop/case_handoff tools; (2) storage-integrity enum validation (case_type/priority in case-tools.js, CASE_TYPE_VALUES/PRIORITY_VALUES) which guards AFTER the model has already exercised judgment via its tool call, catching a malformed value before it corrupts an observability aggregate -- not standing in for interpretation. Every field a read-only observability view (map, SLA-by-type, workload-by-type) needs is agent-settable through the normal case_* tool flow (case_type via case_update, lat/lon via case_report) -- set autonomously as the agent gathers the report, never left for a human to classify by hand afterward. Only genuinely operator-only fields (autonomy mode) stay human-set, because flipping it back to auto would let the agent escape the very mode a human used to stop it.
