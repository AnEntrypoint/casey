gm-method (witnessed): when live-witnessing a language/locale tie-break fix (e.g. two closely-related languages sharing several cue tokens) with a hand-written test string, an unexpected non-tie result can mean the test string accidentally contains a real distinguishing marker for one of the two languages rather than the code being wrong -- before concluding a bug, manually recompute each branch's actual score/hit-list against the exact test string and confirm which cues fired, rather than trusting the intended-ambiguous framing of the test case.
