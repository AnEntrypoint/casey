gm-method (witnessed): when a user reports the SAME class of defect three times in a row and each fix was 'add the missing keyword/phrase', that is the tell that you are patching symptoms of a wrong ARCHITECTURE, not fixing the bug. A fixed keyword/phrase detector can never cover an open-ended natural-language space; each patch just moves the gap. The durable fix is to replace the hard classifier with INTERPRETED intent (let the in-loop model read the message into a structured intent, with a small SHAPE-based deterministic fallback that SUGGESTS not dictates) and SOFT state transitions (a state machine whose guards FLAG rather than BLOCK), so the structural dead-end (one branch that closes out anything it does not recognize) becomes impossible by construction instead of guarded against per-phrase. Lesson: count the repeats -- 3+ same-class user reports means reshape the approach, don't extend the list. Verify the reshape with a classifier matrix over true-positives AND must-not-trip content, plus a real-services end-to-end that the old dead-end output never appears.
