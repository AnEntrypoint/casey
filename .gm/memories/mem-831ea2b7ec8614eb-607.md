gm-method (witnessed): a single value used as BOTH a conversation/session IDENTITY key AND the message DELIVERY target conflates two concerns and breaks multi-party channels. A group channel id keyed the session, so every participant shared one session; but the reply had to go to the channel, so the id could not simply be swapped for the per-user id. Fix: split them -- identity = container+author (per participant), delivery = the container. Audit every use of the old combined value and route each to identity or delivery. Symptom: one user's action attaching to another user's session in a shared room.
