gm-method (witnessed): when a conversational system must CAPTURE structured fields from free-form natural-language user replies and a brittle regex misses a common phrasing (causing a re-ask loop), do NOT just widen the regex -- build STACKED, INDEPENDENT, MODEL-DOWN-SAFE floors so the answer always binds: (1) a broadened deterministic fast-path (allowlist-BY-SHAPE -- accept an unknown proper-noun-shaped token, reject known common/colliding words -- not a fragile denylist); (2) an LLM structured-extraction floor (a cheap FORCED single-tool call interpreting the message into the field vocabulary, merged FILL-IF-EMPTY UNDER the deterministic result so the cheap path wins on its closed vocabulary and the model fills the long tail); (3) a pending-ask binder that binds the reply to the field just asked; (4) a TOTAL asked-marker so EVERY ask path (incl model-prompted prose) leaves a pending key. Each floor catches what the one above missed; with the model absent every floor degrades to the next, so it never dead-ends. Critical guards the adversary catches: the extraction floor must run ONLY on confirmed intake (gated AFTER enquiry/intent routing) or a query-target place gets written as a field; exclude PII fields (a phone) from the extraction whitelist and the summary projection; run the new-case-conflict check on the UNION of regex+LLM fields so a newly-extracted differing value still branches; and a forced side-tool call must not corrupt a model that counts its own invocations -- answer it without advancing turn state.
