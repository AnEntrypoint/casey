gm-method (witnessed): a prd-add row's witness text alone does not satisfy the gate -- omitting the status:"done" field (or using a status value the gate's ready_wave filter does not recognize as terminal) leaves the row silently pending even with a complete witness written. residual-scan/CONSOLIDATE will report "PRD still has items" with no obvious diff; the fix is to re-dispatch instruction and read ready_wave directly (the authoritative pending list) rather than hand-filtering a full prd-list dump against a guessed status vocabulary -- different sessions' historical PRD stores can use different status strings ("completed" vs "done"), so ready_wave from a fresh instruction call is the only reliable read of what the gate currently considers pending.
