gm-method (witnessed): in a slot-filling / intake dialogue, EVERY question the system can ask must have a capture path for its answer, or it loops forever. A field that is ASKED but has no extractor (and the model is too weak to record it) drops the user's reply, the 'next question' selector re-derives the same gap, and the identical question repeats. Two-part fix: (1) record durably WHICH field each question asked, and select the next question by skipping already-asked fields (so no field repeats); (2) a pending-ask binder that attaches the next free-text inbound to the field last asked when nothing else captured it. Symptom to watch for: the same bot question appearing twice in a row in a transcript -- it always means an ask whose answer has nowhere to land.
