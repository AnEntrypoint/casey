gm-method (witnessed): a conversational/agent system can route an INFORMATIONAL QUERY ('what's on my list today?') down the same path as CONTENT INTAKE, because the only handler for the query is an LLM tool-call the weak production model never emits -- so the query silently falls through to the intake/complete path and gets a wrong canned reply. The fix is a DETERMINISTIC detector that short-circuits the query BEFORE the model turn (mirroring an existing fixed-keyword short-circuit), plus a content VETO list so genuine content is never misclassified as a query. Lesson: when a capability exists only as an LLM tool but the production model is too weak to call it, the deterministic path must own the routing, not the model. Verify by an exec_js classifier matrix over BOTH true-positive and must-be-null content cases, then a real-services end-to-end assertion that the reply is the new shape and not the old canned exit.
