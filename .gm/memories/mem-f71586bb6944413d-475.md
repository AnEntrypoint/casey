casey fact-base caveat: thatcher persists event.data as a JSON STRING and store.listEvents returns it UNPARSED, so any consumer reading data.from/data.to/data.by/data.field must JSON.parse first. The shared parser is overview.js evData() (reused by workload.js and report-analytics.js); the dashboard parses at the /api/cases/:id + /events boundary via parseEventData so the SPA always receives object data. Forgetting to parse silently yields undefined fields, not an error.
