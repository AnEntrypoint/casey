casey dashboard/report open-case filtering: src/format.js exports isOpenCase(c) (status !== 'resolved' && status !== 'closed') as the single source of truth. Before 2026-07-10 several dashboard endpoints (clusters, geo, map, attention/inbox, sla-at-risk, handover, unreplied, suggestions) filtered only status!=='closed', inconsistent with report.js which already excluded resolved too -- a resolved outbreak kept showing as an active map pin. All call sites in src/dashboard/server.js and src/report.js now use isOpenCase. Any NEW open-case filter in this codebase should import isOpenCase from format.js rather than reimplementing the status check. Fixed 2026-07-10, commit 4cd7784.
