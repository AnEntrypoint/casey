gm-method (witnessed): when a request matches an existing purpose-built skill exactly (e.g. a "maximize conversational quality across the whole system" ask matching a repo's own casey-maximize-quality skill), invoking that skill directly and letting its internal workflow (map -> per-surface audit -> adversarial default-refute verify -> synthesize) drive the fan-out is strictly better than hand-authoring an equivalent Workflow script inline -- it reuses tuned prompts/lenses and stays consistent across sessions. After the workflow returns a ranked, already-verified fix plan, apply every confirmed finding directly (Edit) rather than re-deriving or re-verifying them -- the workflow's adversarial verify step already did that work; re-checking is only needed live (node --check, a smoke test exercising the actual changed behavior) to confirm the applied patch matches the plan's intent, not to re-litigate whether the finding was real.
