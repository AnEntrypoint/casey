Latent-bug class: a pure (data, now) helper that parses a timestamp with Date.parse() passes its ISO-string unit tests yet silently returns 0/NaN on live rows when the system-of-record persists timestamps as numeric unix-SECONDS (Date.parse(number) coerces to string -> NaN). The witness is a live-store probe of the actual field type, not the unit test -- a mock-free integration assertion that exercises the REAL persisted shape catches it; the ISO-string unit test never can. Lesson: when a helper is shared between a test surface (ISO strings) and a live surface (numeric epoch), normalize at the parse edge (number<1e12 -> *1000 else Date.parse) and witness with a real-services row read, not just the pure unit path.
