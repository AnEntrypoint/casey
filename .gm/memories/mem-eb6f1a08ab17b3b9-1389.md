gm-method (witnessed): when a conversational/NL system's failures are being fixed by repeatedly ADDING keywords/regex/typo-tolerance/phrase-tables (whack-a-mole), that is the tell the ARCHITECTURE is wrong, not the individual bug -- a fixed classifier can never cover an open-ended NL space. The durable fix is to REMOVE the deterministic text-processing layer entirely and let the LLM agent classify + extract + route + answer via TOOLS (the agent calls tools, tools return data, the model relays), keeping ONLY: (a) a tiny deterministic safety short-circuit for IRREVERSIBLE controls (opt-out/handoff) that must fire even model-down, (b) a deterministic EMPTY-CASE FLOOR that records plainly-stated facts fill-if-empty so a message is never silently dropped when the weak model calls no tool, (c) outbound scrubs. On an LLM outage, QUEUE the message + re-drive on recovery (a durable marker + a recovery-edge sweep), never a fallback classifier.
