gm-method (witnessed): a multi-agent quality-audit workflow's proposed fix can itself contradict a codebase's own stated architectural invariant -- verify each finding against a DIRECT read of the surrounding code and its documented design principle before applying, not just against the finding's own reasoning. Concretely: a workflow flagged 'tool X bypasses the observe-mode guard other tools have' as a bug, but the codebase's own comments established that certain controls (irreversible/legal, e.g. opt-out or human-handoff) are DELIBERATELY exempt from a content-edit guard and must fire regardless of mode -- applying the proposed fix verbatim would have re-introduced the exact bug a sibling fix in the same batch was correcting (the deterministic STOP/HUMAN path was just fixed to fire in EVERY mode). The correct move was re-scoping the PRD row from 'apply the patch' to 'read the code, confirm intent, document it clearly instead' -- a workflow's severity/confidence framing does not override direct verification against the target codebase's own documented invariants.
