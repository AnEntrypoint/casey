gm-method (witnessed): to find the FULL set of misroutes in an NL intent classifier (not just the one a user reported), fan out subagents to ENUMERATE a large matrix of natural real-user phrasings per intent (20-40 each, with typos/code-switching/fragments), then CLASSIFY each against the REAL classifier source (pass the actual file to the agents so they trace the real regexes, not imagine them) and flag every case where the route deflects/wrong-routes despite a real answer existing. One reported failure ('where are we at') expanded to 161 misroutes across 334 phrasings -- the matrix surfaces the systemic gap a single example hides. Implement the synthesized fixes, but re-run the WHOLE existing matrix after each batch: broadening one intent's triggers routinely steals another's ('how is my report'=status vs 'how are we doing'=overview collided; a tolerant ref regex over-matched a trailing word). Two recurring traps when adding NL triggers: (1) a possessive-singular vs plural distinction ('my report'=status-of-mine vs 'my cases'=list) needs an explicit guard; (2) an optional-suffix regex with a space-separator greedily swallows the next word -- anchor the suffix to a hyphen/attached form. Always keep the safe-default veto (a domain report stays a report) checked FIRST so no new trigger can swallow the primary intent.
