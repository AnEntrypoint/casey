casey src/gateway-hooks.js detectContactIntent: HUMAN_KEYS' bare single-word 'human' is gated by AMBIGUOUS_HUMAN_KEYS + AMBIGUOUS_MAX_WORDS (same discipline as AMBIGUOUS_STOP_KEYS for STOP) so a report sentence like 'the human gave it water' does not false-positive a handoff. Exclude-phrase matching (STOP_EXCLUDE/HUMAN_EXCLUDE) is scoped via excludedAt()/EXCLUDE_WINDOW to a token window around the SPECIFIC matched key occurrence, not the whole message -- a genuine handoff request elsewhere in a longer message is no longer suppressed by an unrelated excluded phrase.
