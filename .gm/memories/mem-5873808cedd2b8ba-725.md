gm-method lesson: when a dependency primes a process-global singleton (e.g. a sqlite handle) lazily from the CURRENT WORKING DIRECTORY at first call, re-importing its accessor to point it elsewhere forks the module graph into a SECOND independent handle -- the relocation silently does not take. The only safe relocation is the process cwd itself. So an integration test that must not clobber a live data store copies the config into a temp dir and chdir's there before importing the dependency, keeping any sibling `file:../` imports anchored to the original repo root. Generalizes to any cwd-bound or import-time-initialized global resource: relocate by changing cwd before first import, never by re-importing the accessor.
